默契挑战 App 需求文档（双平台 + 平板/手机兼容）

1. 项目概述
   项目名称：默契挑战。

目标平台：Android + iOS（手机 + 平板全面兼容）。

核心玩法：支持 2-10 人在线进行默契问答、你画我猜等挑战，测试玩家之间的熟悉程度和配合度。

目标用户：朋友聚会、团建、情侣互动场景，年龄 16-35 岁。

网络模式：在线房间制，玩家通过房间号加入同一场游戏。

1. 功能需求
   2.1 游戏模式
   2.1.1 默契问答
   玩法：

系统随机选择一名玩家作为出题者（轮流担任），其余玩家同时回答关于出题者的问题。

问题分为两类：预设题库（如“我最喜欢的颜色”“我的生日月份”）和自定义问题（由出题者现场输入）。

玩家答案与出题者的真实答案一致时，该玩家得分。

计分规则：

每道题答对得 1 分。

出题者不得分，但所有玩家答完后出题者公布真实答案。

题库管理：

预设题库不少于 100 组，支持用户自定义选取。

自定义问题可由出题者在本轮开始前输入，其他玩家不可见。

流程：

系统显示问题，所有猜题者限时 30 秒作答（可选择多个选项或文本输入）。
倒计时结束或所有人提交后，出题者点击“揭晓答案”。
系统展示各玩家答案与真实答案的匹配结果，并累计得分。
进入下一轮，换下一人做出题者。
轮次：默认进行 3 轮（可配置），结束后进入最终排行榜。

2.1.2 你画我猜（接力版）
玩法：

系统随机选择一名玩家作为当前作画者，其余玩家为猜词者。

作画者从题库中抽取一个词（展示四个词，选取一个），可以选择跳过换一批词。限时 80 秒在画板上作画。

猜词者随时在聊天框输入答案，系统按提交时间顺序记录猜对者。

一轮结束条件：80 秒倒计时结束，或所有猜词者都已猜对。

计分规则：

猜词者得分（按猜对顺序）：

第 1 个猜对：+3 分

第 2 个猜对：+2 分

第 3 个猜对：+1 分

第 4 个及以上猜对（若人数 >4）：+1 分

作画者得分：

基础分：本轮有至少 1 人猜对时，作画者得 +2 分；无人猜对得 0 分。

额外加成：每有 1 人猜对，额外 +1 分。

总分 = 2 + (猜对人数)（当猜对人数≥1），最多四分。

多人同时猜对（时间戳相同）时，按同一顺序位计算得分，例如 2 人同时第一个猜对 → 各得 3 分，下一个猜对者视为第 3 个，得 1 分。

题库：

内置词库不少于 100 个，涵盖名词、成语、动作等，可扩展。

词库按难度分级（简单、中等、困难），作画者可自行选择或随机抽取。

画板功能：

支持多种画笔颜色、粗细调节，橡皮擦，清屏。

支持撤销/重做（至少 5 步）。

作画过程实时同步给所有猜词者（低延迟）。

特殊规则：

作画者可在任意时刻点击“跳过”换词（最多 2 次），跳过不计时，新词重新计时。

猜词者不可重复提交相同答案（同一轮对同一词只记录第一次）。

2.1.3 （可选）动作同步
作为后续版本功能，本次 MVP 暂不实现，但预留接口。

2.2 房间与匹配
创建房间：

房主设置游戏模式（默契问答 / 你画我猜）、最大人数（2-10）、轮次数（默契问答）、作画时间（你画我猜）等。

生成 6 位房间号，玩家通过房间号加入。

加入房间：

输入房间号，若房间未开始且人数未满则可加入。

每个玩家在加入时可选择昵称（默认随机生成）。

开始游戏：

房主点击“开始游戏”，游戏开始后不可再加入新玩家。

若游戏模式为你画我猜，随机确定作画顺序（房主可重排）。

实时同步：

使用 WebSocket 实现状态同步（玩家进出、游戏开始、作画数据、答案提交等）。

网络不稳定时自动重连，重连后恢复当前游戏状态。

2.3 辅助功能
得分榜：

每轮结束后显示本轮得分和累计总分。

最终游戏结束后显示完整排行榜（前三名高亮）。

游戏音效：

开始倒计时、答对/答错提示音。

支持静音开关（设置中可配）。

新手引导：

首次进入游戏时弹出引导浮层，介绍基本操作。

分享功能：

游戏结束后可生成战绩海报（含得分、MVP、房间号），分享到微信/QQ/系统分享面板。

历史记录：

本地存储最近 10 场游戏的简要数据（时间、模式、参与者、最终得分）。

1. 非功能需求
   响应速度：

界面切换 ≤ 200ms，画板同步延迟 ≤ 300ms。

兼容性：

设备类型：手机（屏幕宽度 320dp \~ 480dp）和平板（宽度 ≥ 600dp）均需完美适配，布局自适应（使用响应式设计）。

操作系统：Android 8.0 (API 26) 及以上，iOS 13.0 及以上。

屏幕方向：手机默认竖屏锁定，平板支持横竖屏切换（推荐横屏体验）。

稳定性：

连续运行无崩溃，内存占用 ≤ 200MB。

安全性：

不收集用户隐私，房间号仅用于临时连接，不存储用户真实身份。

所有通信加密（WSS）。

离线处理：

网络断开时提示用户，尝试重连 3 次，若失败则退出到房间列表并保留游戏进度（10 分钟内可重连）。

1. 用户界面（UI）
   启动页
   主要元素：应用图标、加载进度

交互说明：自动检查更新，显示版本号

平板适配要点：居中放大图标，背景图片可拉伸

首页
主要元素：创建房间、加入房间、设置、排行榜（本地）

交互说明：点击创建房间进入创建界面；点击加入房间弹出输入框

平板适配要点：使用 Column 或 Grid 布局，按钮间距增大，充分利用宽度

创建房间
主要元素：模式选择（默契问答/你画我猜）、人数滑块（2-10）、轮次数/作画时间设置、开始按钮

交互说明：房主可调整参数，点击开始后进入等待界面

平板适配要点：滑块和选项可两列布局，避免过长的单列

等待房间
主要元素：房间号（可复制）、玩家列表、房主可见开始按钮

交互说明：非房主只有玩家列表，显示“等待房主开始”

平板适配要点：玩家列表使用 Wrap 或网格，每个玩家卡片宽度自适应

默契问答游戏页
主要元素：问题文本、选项/输入框、倒计时、当前出题者标识

交互说明：出题者需先输入自定义问题（若选择自定义）后进入答题阶段

平板适配要点：问题文本使用更大字号，选项按钮可水平滚动或网格排列

你画我猜游戏页
主要元素：画板（仅作画者显示画笔工具栏）、猜词输入框（猜词者可见）、倒计时、跳过按钮（作画者）

交互说明：作画者绘画时，猜词者实时观看画板并提交答案

平板适配要点：平板上画板区域可以更大，工具栏放在侧边而非顶部/底部，猜词输入框浮动在画板角落

结算页
主要元素：本轮得分明细、总分榜、下一轮/返回大厅按钮

交互说明：点击返回大厅后重置游戏，回到房间等待状态

平板适配要点：得分明细可分成两列显示（排名、得分），总分榜突出显示

1. 技术栈建议
   客户端
   框架：Flutter（最新稳定版）

语言：Dart

架构：MVVM + Provider / Riverpod

画板：CustomPaint + GestureDetector，点阵数据通过 WebSocket 广播

本地存储：shared\_preferences 或 sqflite

后端（自托管 Supabase）
方案：Supabase（开源，可自托管）

部署位置：用户自己的服务器（需满足 Docker 环境）

核心组件：

PostgreSQL 数据库：存储房间信息、玩家数据、得分记录

Realtime 功能：用于房间状态、画板数据的实时同步（基于 WebSocket）

Auth：匿名登录（无需用户注册）

客户端配置：

客户端代码中，Supabase 的 URL 和 anon key 必须通过配置文件或环境变量读取（例如 lib/config.dart 或 --dart-define），不能硬编码。

Trae 生成的代码应包含一个示例配置文件（如 config.example.dart），用户复制后填入自己服务器的真实地址和密钥。

7\. 后端准备要求（用户自行完成，不在 Trae 生成范围内）
为了客户端能正常联网，你（用户）需要提前在自己的服务器上部署好 Supabase（自托管）。部署完成后，你会得到两个关键信息：

Supabase URL（例如 <http://你的服务器IP:8000> 或域名）

anon key（一串字符串）

Trae 生成的 Flutter 代码会预留配置文件（如 lib/config.dart 或 .env），你需要把上述两个值填入。

快速部署命令（在你的服务器上执行）
bash

# 安装 Docker 和 Docker Compose（如果尚未安装）

# 克隆 Supabase 自托管仓库

git clone <https://github.com/supabase/supabase>
cd supabase/docker

# 复制环境变量模板

cp .env.example .env

# 编辑 .env，设置以下关键项：

# POSTGRES\_PASSWORD=你的强密码

# JWT\_SECRET=你的JWT密钥（生成一个随机字符串）

# ANON\_KEY=公开密钥（可自动生成，但建议填写固定值）

# SERVICE\_ROLE\_KEY=服务角色密钥

# 启动服务

docker-compose up -d
获取客户端配置
Supabase URL：<http://你的服务器IP:8000（如果使用反向代理则用域名，建议配置> HTTPS）

anon key：从 .env 中的 ANON\_KEY 获取

数据库表结构（需提前创建）
客户端代码会依赖以下表，用户需在 Supabase 中创建（可通过 SQL 编辑器执行）：

rooms：房间信息（房间号、模式、状态、创建时间等）

players：玩家信息（昵称、房间号、得分等）

game\_sessions：游戏会话（当前轮次、题目、作画数据等）

draw\_actions：画板点阵历史（用于重连恢复）

具体的 SQL 建表语句可由 Trae 生成代码时一并提供（如 supabase\_schema.sql），用户导入即可。
